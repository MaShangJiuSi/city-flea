## 1. 登录码是什么？

### 1.1 系统设计理念

同城跳蚤市场系统设计为**微信小程序授权登录**，但在开发阶段，我们提供了**模拟登录**机制。

### 1.2 登录码（code）到底是什么？

`登录码` = **微信小程序的 js_code**

**真实流程**：
```
用户在小程序点击登录
    ↓
小程序调用 wx.login() 获取 js_code
    ↓
发送 js_code 到后端
    ↓
后端调用微信 API 换取 openid
    ↓
返回 JWT Token
```

**开发环境模拟**：
```
用户在前端输入任意字符串作为"登录码"
    ↓
后端直接创建测试用户
    ↓
返回 JWT Token
```

### 1.3 为什么需要登录码？

- **生产环境**：需要微信小程序的真实 AppID 和 AppSecret
- **开发环境**：使用模拟码绕过微信验证

## 6. 常见问题

### Q1：登录失败，提示"登录失败"？

**原因**：微信 API 调用失败（没有配置真实的 AppID）

**解决方案**：
- 使用模拟登录码（以 `TEST_` 开头）
- 或配置微信测试号的 AppID 和 AppSecret

### Q2：管理员登录失败？

**原因**：数据库中没有管理员账号

**解决方案**：
1. 导入测试数据：`test_data.sql`
2. 或手动执行：
```sql
INSERT INTO `employee` 
(`Name`, `username`, `password`, `phone`, `sex`, `id_number`, `status`)
VALUES 
('管理员', 'admin', '123456', '13800138000', '1', '110101199001011234', 1);
```

### Q3：页面没有数据？

**检查清单**：
1. ✅ 数据库是否已启动
2. ✅ 测试数据是否已导入
3. ✅ 后端服务是否运行在 8080 端口
4. ✅ 前端服务是否运行在 5173 端口
5. ✅ 数据库是否名为 `city_flea`

**验证步骤**：
```bash
# 检查数据库
mysql -u root -proot city_flea -e "SHOW TABLES;"

# 检查数据
mysql -u root -proot city_flea -e "SELECT COUNT(*) FROM goods;"
```

### Q4：如何配置真实的微信登录？

#### 步骤 1：申请微信测试号

访问：https://mp.weixin.qq.com/debug/cgi-bin/sandbox?t=sandbox/login

#### 步骤 2：获取测试信息

获取：
- AppID
- AppSecret

#### 步骤 3：配置后端

编辑 `application.yml`：
```yaml
wechat:
  appid: wx1234567890abcdef
  secret: 1234567890abcdef1234567890abcdef
```

#### 步骤 4：使用真实 js_code

1. 在微信公众平台测试号页面获取 js_code
2. 在前端输入该 js_code
3. 后端会调用微信 API 完成登录

### Q5：前端页面空白？

**可能原因**：
- API 请求失败
- CORS 跨域问题

**解决方案**：
1. 检查浏览器控制台错误
2. 确认后端是否正常运行
3. 检查 API 地址配置（应为 http://localhost:8080）
