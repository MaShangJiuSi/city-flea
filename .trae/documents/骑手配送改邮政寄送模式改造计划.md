# 同城跳蚤市场系统 - 骑手配送改邮政寄送模式改造计划

## 一、项目概述

### 1.1 改造目标
将现有骑手即时配送模式改造为**邮政/快递寄送模式**，通过对接快递鸟/菜鸟物流API实现运单创建、物流轨迹同步、运费计算等功能，同时保留微信支付能力。

### 1.2 核心改动
- **订单流程**：骑手接单配送 → 卖家发货→ 创建运单 → 物流追踪
- **物流集成**：对接快递鸟/菜鸟API，支持多快递公司
- **运费计算**：按收货区域自动计算运费
- **代码处理**：骑手模块保留但禁用，配置开关控制

## 三、改造方案

### 3.1 目标订单流程（改造后）
```
待支付(1) → 待发货(2) → 已发货(3) → 物流运输中(4) → 待收货(5) → 已完成(6)
                              ↓
                           已取消/已退款
```

### 3.2 新增/修改的订单状态
| 状态码 | 名称 | 说明 |
|--------|------|------|
| 1 | 待支付 | 订单创建，等待付款 |
| 2 | 待发货 | 支付成功，等待卖家发货 |
| 3 | 已发货 | 卖家已发货，运单已创建 |
| 4 | 物流运输中 | 物流状态同步中 |
| 5 | 待收货 | 快递已送达，等待买家确认 |
| 6 | 已完成 | 买家确认收货 |
| 7 | 已取消 | 订单取消 |
| 8 | 退款中 | 退款处理中 |
| 9 | 已退款 | 退款完成 |

## 四、具体改造内容

### 4.1 数据库改造

#### 4.1.1 修改 orders 表
```sql
ALTER TABLE orders
  -- 删除骑手相关字段（保留但置空）
  DROP COLUMN rider_id,
  DROP COLUMN take_time,
  -- 新增物流相关字段
  ADD COLUMN express_company VARCHAR(32) COMMENT '快递公司编码',
  ADD COLUMN tracking_number VARCHAR(64) COMMENT '运单号',
  ADD COLUMN ship_time DATETIME COMMENT '发货时间',
  -- 保留 delivery_fee 作为运费字段
```

#### 4.1.2 修改 order_delivery_track 表
```sql
ALTER TABLE order_delivery_track
  DROP COLUMN rider_id,
  ADD COLUMN express_status VARCHAR(32) COMMENT '快递状态',
  ADD COLUMN location VARCHAR(255) COMMENT '物流位置',
  CHANGE COLUMN track_status track_status INT COMMENT '0系统 1快递轨迹';
```

#### 4.1.3 新增运费配置表
```sql
CREATE TABLE freight_config (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  province_code VARCHAR(16) COMMENT '省份编码',
  province_name VARCHAR(32) COMMENT '省份名称',
  first_weight DECIMAL(10,2) COMMENT '首重价格(元)',
  continued_weight DECIMAL(10,2) COMMENT '续重价格(元/500g)',
  enabled TINYINT DEFAULT 1 COMMENT '是否启用',
  create_time DATETIME,
  update_time DATETIME
) COMMENT '区域运费配置表';
```

#### 4.1.4 新增快递公司配置表
```sql
CREATE TABLE express_company (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  code VARCHAR(32) COMMENT '快递公司编码',
  name VARCHAR(64) COMMENT '快递公司名称',
  logo VARCHAR(255) COMMENT 'Logo URL',
  sort INT DEFAULT 0 COMMENT '排序',
  enabled TINYINT DEFAULT 1 COMMENT '是否启用',
  is_default TINYINT DEFAULT 0 COMMENT '是否默认',
  create_time DATETIME,
  update_time DATETIME
) COMMENT '快递公司配置表';
```

#### 4.2.2 骑手模块处理策略

**策略：保留但禁用，通过配置开关控制**

1. **配置文件** (`application.yml`)
   ```yaml
   city:
     delivery:
       mode: express  # express=快递模式, rider=骑手模式(已废弃)
       enabled: false # 骑手模块是否启用，默认关闭
   ```

2. **拦截器处理**
   - `JwtTokenRiderInterceptor.java`：检测配置，若骑手模式关闭，返回"骑手端已停用"

3. **数据库表**
   - `rider` 表、`order_delivery_track` 表保留，不做删除

4. **代码注释**
   - 在 `RiderService`、`RiderController` 等类上添加 `@Deprecated` 注解
   - 添加注释说明骑手模块已迁移至快递配送模式

### 4.3 运费计算系统

#### 4.3.1 运费计算逻辑
```
总运费 = 首重价格 + ceil((商品重量 - 500g) / 500g) × 续重价格
```

#### 4.3.2 商品关联重量
- 在 `goods` 表新增 `weight` 字段（克为单位）
- 默认重量（如500g）可通过配置设置

#### 4.3.3 运费查询接口
- 根据收货地址自动匹配运费配置
- 返回可用快递公司和预估运费

## 九、风险与注意事项

1. **快递鸟API限制**：免费接口有次数限制，生产环境需购买服务
2. **物流状态同步延迟**：第三方推送可能有延迟，需做好兜底轮询
3. **数据库迁移**：生产环境需做好数据备份和回滚方案
4. **兼容旧订单**：已存在的骑手配送订单需特殊处理
